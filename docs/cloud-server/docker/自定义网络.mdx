# 自定义网络

在 Docker 中,默认的网络模式是 `bridge`,Docker 会为每个容器分配一个唯一的 IP 地址。然而,容器的 IP 地址可能会因为重启或其他原因发生变化,这会导致容器之间的通信变得不稳定。为了解决这个问题,Docker 提供了自定义网络功能,通过自定义网络,可以使用容器名作为稳定的域名进行通信。





## 创建自定义网络

使用 `docker network create` 命令可以创建一个自定义网络。

```bash
docker network create 自定义网络名
```



## 使用自定义网络

在运行容器时,可以通过 `--network` 参数指定容器加入的自定义网络。

```bash
docker run -d --name 容器名 --network 网络名 镜像名
```
