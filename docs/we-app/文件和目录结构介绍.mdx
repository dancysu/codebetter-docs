# 文件和目录结构介绍

## 文件

一个完整的小程序项目分为两个部分：主体文件、页面文件。

### 主体文件

主体文件又称全局文件,能够作用于整个小程序,影响到小程序的每个页面,主体文件必须放到项目的根目录下。

```bash
.
├─ app.js # 小程序入口文件
├─ app.json # 全局配置
└─ app.wxss # 全局样式表
```

小程序的全局结构由以下三个关键文件组成,它们都位于项目的根目录：

- `app.js`：作为小程序的入口文件,**是必须的**。
- `app.json`：用于全局配置,**是必须的**。
- `app.wxss`：定义小程序的全局样式。

### 页面文件

页面文件是 每个页面所需的文件,小程序页面文件都存放在 pages 目录下,一个页面一个文件夹。

```bash
.
└─ pages
   └─ index # index页面
   	  ├─ index.js # 页面逻辑
   	  ├─ index.json # 页面配置
   	  ├─ index.wxml # 页面结构
      └─ index.wxss # 页面样式表
```

一个小程序页面由四个文件组成,每个文件仅对当前页面有效,分别是：

- `.js`：包含页面的逻辑处理代码,**是必需的**。
- `.wxml`：定义页面的结构,**是必需的**。
- `.wxss`：页面的样式表,用于美化页面。
- `.json`：页面的配置文件,用于设置页面特定的配置。页面中配置项会覆盖 `app.json` 的 `window` 中相同的配置项。



## 目录结构

```bash
.
├─ components # 公共组件目录
├─ pages # 页面目录
├─ .eslintrc.js
├─ app.js
├─ app.json
├─ app.wxss
├─ project.config.json # 开发者工具的配置  
├─ project.private.config.json
└─ sitemap.json
```
